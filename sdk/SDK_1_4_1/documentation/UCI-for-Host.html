UWB Command Interface (UCI)

The UWB Command Interface (UCI) enables communication with the Ultra-wideband subsystem (UWBS). Each message should obey the packet format depicted in Figure 1. The initial four bytes define the type and length of the consecutive payload. It will also indicate whether a Cyclic redundancy check (CRC) is included at the end.

Figure 1. UCI Packet Format

Each UCI Packet shall have a 4-octet Packet Header (see Figure 2). The contents will be laid out below.

Figure 2. UCI Packet Header Format

Message Type (MT)

The MT field indicates the contents of the Packet and shall be a 3-bit field. The following values are defined: a Command message (CMD, 0x01) which provides instructions to the UWBS, a Response message (RSP, 0x02) that is always a (direct) response following a command, and a notification message (NTF, 0x03) that informs about some state which can occur at any time.

Packet Boundary Flag (PBF)

The PBF bit is used for Segmentation and Reassembly. If its value equals 1, the Packet will contain only a segment of the payload. If it equals 0, it will contain a complete message, or the Packet contains the last segment of the segmented message. Each segment shall have the same MT, GID and OID values.

Group Identifier (GID)

The UCI will use 4-bit instruction opcodes that are categorized according to their individual groups .

The GID of the Device UWB subsystem equals 0x0.

The GID of the Configuration UWB subsystem equals 0x1.

The GID of the Actions UWB subsystem equals 0x2.

Opcode Identifier (OID)

The Opcode Identifier (OID) indicates the identification of the Control Message and shall be a 6-bit field that is a unique identification of a set of Command/Response Messages within the group. OID values are defined along with the definition of the respective Control Messages described in Table 31.

CRC Enable (CE)

The C-bit in the Packet Header indicates whether the payload is extended with 2 additional bytes. This extension represents the CRC16 outcome of the Header and Payload (4+L bytes) which can be used to check for Packet transfer errors.

Payload Length (L)

The Payload Length shall indicate the number of octets present in the payload. The Payload Length field shall be a 10-bit field, representing values 0 to 1023.

Device state
Value Label / Description
0x01 DEVICE_STATE_READY. - UWB is initialized and ready to begin. This state is the first state after power ON. In this state, the device shall accept all Device and Configuration commands.

0xFF DEVICE_STATE_ERROR. - An unrecoverable error occurred. A reset is required.

0x4 DEVICE_STATE_REASON_HPD_RESET_PIN. - Reset pin during hard power down was the reason for the reboot.
0x5 DEVICE_STATE_REASON_HPD_WUP_PIN. - Wake up pin during hard power down was the reason for the reboot.

0x21 DEVICE_STATE_REASON_DPD_GPIO. - Wake up gpio event during hard power down was the reason for the reboot.
0x24 DEVICE_STATE_REASON_INVALID_STS_CONFIG. - The Scrambled Timestamp Sequence (STS) is not configured correctly.

Set persistent reg

Prototype: SetPersistentRegRsp* setPersistentReg(uint32_t value)

Description: To retain 32 bits of data in RAM that survive a device reset. When the device is resetted using the reset pin / button, resetted via software, or awakens from a deep or hard power down, all data in RAM is cleared. However, there are 4 bytes of data that persist and survive a reset. This can be set as a 32 bit value using this function. To read back the data, use getPeristentReg(). These 32 bits are cleared only after a Power-On Reset.
